App Developer Guide

Atoms

Atoms are the building blocks. This is where we set up the following:

  • Colours
  • Typography
  • Blockqoute
  • Form Items
  • Buttons
  • Images
  • Social media buttons
  • Tables

The sass files for these items are in the styling/scss/atoms folder with the exception of the colours, which are just variables in styling/scss/helpers folder (_site-variables.scss)

To change any of the atoms, go to styling/app-files/atoms and edit the appropriate html file

Molecules

Simple, small components that can either work as self contained components, or as part of a larger organism. Pagination, teaser lists, promoted content blocks are all examples of molecules

Creating a molecule

  • Create an empty html file and save it in styling/app-files/molecules
  • Create your markup in the file. There is no need to create wrappers, or containers here as they will be covered when you inject them into the page. All you are adding here is the basic markup for the block itself. Save and close
  • If you want this file to be accessed via the style app you will need to add a reference to it in the molecules json file. This is at styling/app-files/js/json/molecules.json. Simply add the entry to the bottom. Create a url for it (eg #module-url), text (eg mynewmodulename) and a short description.
  • Finally add a reference to it in the angular app. styling/app-files/js/application.js. Find the molecules list and add it in as follows:

    .when('/mynewmodulename', { templateUrl : 'app-files/molecules/module-name.html', })

    Replacing the references where appropriate

  • You molecule will now be accessible via the apps nav menu. You should then use this page to develop and style your module as a single self contained component. Each module should have its own sass file and js file, and they should share the same names to make it easy to identify what is related to what. so mymodule.html, mymodule.scss, mymodule.js. A lot of the styling may be covered by the atoms, so the styling needed will just be an extension of the existing styling.

Components

Components are similar to molecules, but they are generally a more complicated and larger components, sometimes they are a combination of smaller molecules that form a component. Main navigation, tabbed content and the homepage latest news cluster are examples of an organism.

An example of where an organism is a collection of modules could be the footer area. This may be simply three lists combined to form a 3 column bootstrap layout

Creating an organism is the same as a molecule except the naming will be organism rather than module

A sass file will probably be needed, but not on every occasion, as if it is a collection of modules, then the styling will be covered by the modules.

Templates

Templates are a collection of all of the above, and any coding should be completed by now. You are pretty much just adding includes to a blank html file. Generally there will be 3 parts, the header, content and footer, wrapped in a page-wrapper class:

Templates should not contain actual content or real images. The aim is create very fast prototypes that can be changed and adapted, like a advanced form of wireframing. It is here that ideas can be signed off or rejected

At this stage there should be very little or no component styling required. Its just a case of pasting in and perhaps trying different molecules or organisms. More a case of playing with what should appear on the page and in what position

You will need to add any new templates to the app as per the above process.. create an html file, add to the json, add to the angular application.js file

Header snippet

<div class="page-wrapper"> <!--wraps the entire page -->
<div class="header-wrapper"> <!-- wraps the header -->
<div class="container-fluid full-width-container"> <!--every area needs to be in a container-->
<div class="row">
<section> <!--sections separate the includes -->
<div ng-include="'include file here'"></div> <!--include the molecule or organism -->
</section>
</div>
</div>
</div>

Main content snippet

<!--central content area-->
<div class="content-wrapper"> <!-- wrap the central content -->
<section class="blockmodule"> <!-- blockmodule. Important. This ensures the content doesnt fill the whole width -->
<div class="container-fluid">
<div class="row">
<div class="col-sm-12 col-md-8"> <!--col-md-12 ensures nice padding on mob -->
<section class="main">
<div ng-include="'app-files/organisms/article.html'"></div> <!--include the article molecule -->
<div ng-include="'app-files/atoms/sharing-buttons.html'"></div> <!--include the sharing buttons -->
</section>
</div>
<div class="col-sm-12 col-md-4"><!-- prep the sidebar-->
<aside class="main-sidebar sidebar">
<div ng-include="'app-files/organisms/four-block-news.html'"></div> <!--include the four news block organism -->
</aside>
</div>
</div>
</div>
</section>
</div>

Footer snippet

<!--footer-->
<div class="container-fluid full-width-container footer">
<div class="row">
<section class="blockmodule">
<div>
<div ng-include="'app-files/organisms/footer.html'"></div> <!-- Include the footer organism -->
</div>

</div>
</div>
</div>

Pages

We can now start adding real content and images where possible. There is no individual styling at this stage. Its just a case of copying the templates to the pages folder

You will need to add any new templates to the app as per the above process.. create an html file, add to the json, add to the angular application.js file

When adding the real content, this will mean adding real content to the molecules and components. Now we dont want to add real content to the base ones, as this will affect the templates, and it will be a real pain when reusing the components for future projects. So the molecules and organsims html folders have a production folder within them. Simple copy the components you are using to these folders and edit the content there. All page templates will have the reference to the production "'app-files/organisms/production/article.html'" in the include.
